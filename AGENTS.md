# AGENTS.md

## Проект
Marketing Keyword Automation Platform для site.pro. Полный бриф: `docs/BRIEF.md` —
читай его целиком перед началом КАЖДОЙ новой фазы, даже если уже читал раньше в этой сессии.

Это ВТОРАЯ попытка. Первая реализация была архитектурно правильной (121 тест
проходил, pipeline работал), но деплой ломался многократно из-за мелких,
повторяющихся ошибок. Этот файл существует, чтобы их не повторить.

## КРИТИЧЕСКИ ВАЖНО — порядок работы

Деплой проверяется ПЕРВЫМ, до бизнес-логики, не последним.

**Фаза -1 (обязательная, перед всем остальным):** пустой Yii2 skeleton,
одна страница-заглушка "OK" на главной, сразу Docker, сразу push в GitHub,
сразу деплой на Coolify на реальный публичный URL. Задача — убедиться,
что связка GitHub → Coolify → Docker → публичный URL работает целиком,
ДО написания единой строчки бизнес-логики. Если тут что-то не работает —
чинить пустой skeleton кардинально быстрее, чем систему с 10 сервисами.

После Фазы -1 — переходи к Фазе 0 из `docs/BRIEF.md` и далее по фазам как
описано в брифе.

**После КАЖДОЙ фазы (не только в конце):** закоммить, запушить, дождаться
деплоя на Coolify, вручную (или через curl) проверить, что публичный URL
реально отдаёт ожидаемый результат без 500 ошибок. Не копить несколько
непроверенных на проде фаз подряд — это была главная причина, почему
проблемы всплыли все разом и одновременно на 8-й фазе.

## Известные проблемы предыдущей попытки — НЕ ПОВТОРЯТЬ

1. **cookieValidationKey** должен генерироваться для
   `backend/config/main-local.php` ЯВНО в `entrypoint.sh`. Не полагайся
   на то, что `php init` скрипт сгенерирует его сам — если это не
   сработает (например openssl недоступен при сборке), останется пустая
   строка, и КАЖДЫЙ HTTP-запрос будет падать с 500 из-за
   `InvalidConfigException` в CSRF-защите. Это была причина и "нет CSS",
   и "неверный пароль" одновременно — оба симптома были одним и тем же
   скрытым падением.

2. **Миграции НЕ должны проглатывать ошибки.** Никаких
   `php yii migrate --interactive=0 || echo "non-fatal"` в entrypoint.sh.
   Если миграция падает — контейнер должен упасть явно и видимо в логах,
   не стартовать в сломанном состоянии молча.

3. **i18n файлы** `common/messages/en/app.php` и `ru/app.php` — ПЛОСКИЙ
   массив `['key' => 'value']`. БЕЗ обёртки `['app' => [...]]` —
   `PhpMessageSource` ожидает плоскую структуру, вложенная делает все
   переводы нерабочими без единой ошибки в логах (тихий баг).

4. **GridView импортируется как `use yii\grid\GridView;`.**
   `yii\bootstrap5\GridView` НЕ СУЩЕСТВУЕТ в природе — расширение
   yii2-bootstrap5 не предоставляет свой GridView. Использование
   несуществующего класса — 500 ошибка на КАЖДОЙ странице, где он
   импортирован, без содержательного сообщения в проде (YII_DEBUG=false
   скрывает stack trace).

5. **Каждый созданный файл модели/сервиса** — сразу `git add`. Проверяй
   `git status` на untracked файлы ПОСЛЕ каждой фазы, ДО коммита. Один
   забытый untracked файл модели ломает деплой с непонятной ошибкой
   `Class not found`.

6. **Переключатель языка** должен реально работать: ссылка не может вести
   на `#` без обработчика клика. Либо нормальный `<a href>` на реальный
   route, либо JS, который реально отправляет запрос и обрабатывает ответ.
   Проверь вручную кликом в браузере, не полагайся на то, что код
   "выглядит правильно".

7. **TimestampBehavior + required-правило конфликт:** если модель
   использует `TimestampBehavior` для `created_at`/`updated_at`, эти поля
   НЕ должны быть в `required` правилах валидации — `TimestampBehavior`
   заполняет их в `beforeSave()`, ПОСЛЕ того как `validate()` уже
   отработал и упал бы на required-проверке.

8. **TOCTOU race condition в idempotency-проверках:** если делаешь
   "проверь, есть ли запись → потом вставь" (например, для file_hash
   при импорте) — используй UNIQUE constraint на уровне БД плюс
   insert-with-catch-on-violation, а не check-then-insert в PHP-коде
   без БД-гарантии.

## Правила работы (из исходного брифа, подтверждены)
- Yii2, PHP 8.x, **PostgreSQL** (не MySQL — нужен `pg_trgm` для fuzzy-match)
- `yii\queue` (driver: db), `Codeception` для тестов (не голый PHPUnit)
- SOLID строго: один сервис — одна ответственность, никаких God Objects
- Тесты пишутся ДО контроллеров для каждого сервиса пайплайна
- i18n (en/ru) с самого начала — никаких хардкод-строк в view
- Документация (README) — билингвальная EN+RU, синхронная по содержанию
- Пароль admin — ТОЛЬКО через `ADMIN_PASSWORD` env var с рандомным
  fallback, никогда не хардкодить в миграции
